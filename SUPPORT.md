# Support

Research Ledger is a public preview project. Support is best-effort and focused
on safe, reproducible research-ledger behavior.

## Before Opening An Issue

- Read `README.md`, `SPEC.md`, `docs/threat-model.md`, and `SECURITY.md`.
- Run `research-ledger verify` and copy the exact command output.
- Check whether the problem involves a working-note warning rather than an
  integrity failure.
- Do not upload private research notes, PDFs, snapshots, private keys, or audit
  bundles unless you are certain they are safe to publish.

## Good Issue Reports Include

- Operating system and Python version.
- Research Ledger version or commit.
- The exact command you ran.
- Expected behavior and actual behavior.
- A minimal reproduction using non-private files.
- Whether the ledger was moved between machines or synced through a cloud drive.

## Security Reports

Do not open a public issue with exploit details. Use GitHub Security Advisories
when available. See `SECURITY.md`.

## Methodology Questions

Questions about how to interpret AI-use disclosure, source reliability, legal
evidence, institutional rules, or publication requirements should be treated as
methodology questions, not tool guarantees. Research Ledger can help preserve a
record; it does not decide whether a research process is sufficient.
